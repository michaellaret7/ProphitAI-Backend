# Explain Codebase Section

Analyze and explain: $ARGUMENTS

## Analysis Requirements

1. **Overview**
   - What is the primary purpose of this code section?
   - What problem does it solve?
   - How does it fit into the larger system?

2. **Structure Analysis**
   - List all files/subdirectories with brief descriptions
   - Identify the main entry points and key files
   - Note any important patterns or conventions used

3. **Architecture Diagram**
   - Create a Mermaid diagram showing:
     - Component relationships
     - Data flow
     - Dependencies between files/modules
     - External integrations (if any)

4. **Key Concepts**
   - Explain the 3-5 most important concepts or patterns
   - Describe any non-obvious design decisions
   - Highlight any noteworthy algorithms or approaches

5. **Dependencies & Relationships**
   - What does this section depend on?
   - What depends on this section?
   - How does it interact with other parts of the codebase?

6. **Usage Examples**
   - Show 1-2 concrete examples of how this code is used
   - Include code snippets if helpful

## Output Style

- Use clear, concise language
- Assume the reader is a developer familiar with the tech stack
- Focus on "why" and "how" over "what"
- Use bullet points for scannability
- Include visual diagrams (Mermaid) to illustrate relationships
- Highlight anything surprising or non-obvious

## What NOT to do

- Don't just list files without context
- Don't explain obvious code line-by-line
- Don't include boilerplate descriptions
- Don't make assumptions - if something is unclear, state it
